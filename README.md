# Last Asylum Wiki

A comprehensive game database wiki for Last Asylum, built with Next.js and deployed on Vercel.

All game data is extracted directly from version **1.0.375** using `asylum_ripper.py` and parsed into structured JSON.

## Quick Start

```bash
cd last-asylum-wiki
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Deploy to Vercel

1. Push this folder to a GitHub repo
2. Go to [vercel.com](https://vercel.com), import the repo
3. Vercel auto-detects Next.js — just click Deploy
4. Done. Your wiki is live.


## Updating Game Data

When the game updates:

```bash
# 1. Re-run the ripper to get new bundles
python asylum_ripper.py

# 2. Re-parse the Lua tables
python parse_gamedata.py extracted -o last-asylum-wiki/data

# 3. Commit and push — Vercel auto-deploys
git add -A && git commit -m "Update to version X.X.X" && git push
```

## Adding Images

Copy your extracted sprites into `public/images/`:
```
public/images/
  heroes/           # pic_card4_*.png (hero portraits)
  thumbnails/       # pic_card2_*.png (card thumbnails)
  icons/            # icon_*.png
```

Then reference them in components:
```tsx
<img src={`/images/heroes/${hero.heroIcon}.png`} alt={hero.name} />
```

## Adding Localization

Hero/item display names use string keys that resolve from a localization bundle (`gamedata_basedb_f_en`). Once you extract that bundle, run:

```bash
python parse_gamedata.py extracted -o data --with-locale extracted/gamedata_basedb_f_en
```

This will add `name`, `description` fields to all items/heroes/buildings.
